# 화면 디자인 기준

프로젝트 전반에 일관되게 적용하는 UI/표 디자인 규칙.

---

## 테이블 컬럼 정렬

| 컬럼 유형 | 정렬 | Tailwind 클래스 |
|-----------|------|----------------|
| 날짜, 이름, 티커, 텍스트 | 좌측 정렬 | `text-left` |
| 금액(원화·달러) | 우측 정렬 | `text-right` |
| 비중(%) | 우측 정렬 | `text-right` |
| 변동률(%) | 우측 정렬 | `text-right` |
| 수량 | 우측 정렬 | `text-right` |

**기준:** 수량·금액·비율처럼 크기 비교가 필요한 숫자는 우측 정렬해 소수점이 같은 열에 오도록 한다.

---

## 숫자 표시 형식

| 항목 | 소수점 자리 | 예시 |
|------|-----------|------|
| 원화 금액 (주가, 변동금액 등) | 0자리 (원 단위) | `75,200` |
| 달러 금액 (USD 병기) | 4자리 | `(USD 57.3800)` |
| 비중(%) | 1자리 | `1.2%` |
| 변동률(%) | 1자리 | `+2.5%` |
| 1000개 단위 이하 수량 | 0자리 | `1,234개` |
| 1만개 단위 이하 수량 | 0자리 | `1,234만개` |
| 1억개 단위 이하 수량 | 0자리 | `1,234억개` |
| 1000주 단위 이하 주식수 | 0자리 | `1,234주` |
| 1만주 단위 이하 주식수 | 0자리 | `1,234만주` |
| 1억주 단위 이하 주식수 | 0자리 | `1,234억주` |
| 원 단위 금액 | 0자리 | `1,234원` |
| 만원 단위 금액 | 0자리 | `1,234만원` |
| 10억원 단위 이상 금액 | 0자리 | `1,234억원` |
| 10억원 단위 미만 금액 | 1자리 | `1.2억원` |
| 10조원 단위 이상 금액 | 0자리 | `1,234조원` |
| 10조원 단위 미만 금액 | 1자리 | `1.2조원` |

**원칙:**
- 달러 금액은 항상 원화를 먼저 표시하고 달러를 괄호 안에 부가 표기한다 (CLAUDE.md §5).
- **금액·비율을 포함한 모든 숫자에는 천단위 구분자(,)를 적용한다.** `toFixed()` 단독 사용 금지.

## 숫자 유틸리티 (`lib/fmt.ts`)

모든 화면에서 아래 공유 함수를 import해 사용한다. 페이지 내 로컬 `fmt` 함수 정의 금지.

```typescript
import { fmt, cc, fmtKRW, fmtShares } from "@/lib/fmt"
```

### `fmt(n, dec?)` — 숫자 → 천단위 문자열

```
fmt(75200)        → "75,200"     원화 주가·금액 (dec 기본값 0)
fmt(1.2, 1)       → "1.2"        비중·변동(%)
fmt(12345.6, 1)   → "12,345.6"   억원 (소수 1자리 필요 시)
fmt(null)         → "-"          데이터 없음
```

**dec 기준표**

| 항목 | dec |
|------|-----|
| 원화 주가·금액 (원 단위) | `0` |
| 억원 단위 금액 | `0` |
| 비중(%) | `1` |
| 변동(%) | `1` |
| 달러 (USD 병기) | `4` |
| 수량(주) | `0` |

### `fmtKRW(n)` — 원화 금액 → 만원/억원/조원 단위 문자열

```
fmtKRW(12_000_000_000_000) → "12조원"    (10조 이상, 0자리)
fmtKRW(3_900_000_000_000)  → "3.9조원"   (1조~9.9조, 1자리)
fmtKRW(390_000_000_000)    → "3,900억원" (10억 이상, 0자리)
fmtKRW(1_500_000_000)      → "1.5억원"   (1억~9.9억, 1자리)
fmtKRW(12_340_000)         → "1,234만원" (1만 이상, 0자리)
fmtKRW(500)                → "500원"     (1만 미만)
fmtKRW(-5_300_000_000)     → "-5.3억원"  (음수는 부호 보존)
fmtKRW(null)               → "-"
```

raw KRW 값(원 단위)을 그대로 넘긴다. 사전에 `/1e8` 나누기 금지.

### `fmtShares(n)` — 주식 수량 → 만주/억주 단위 문자열

```
fmtShares(52_000_000)   → "5,200만주"    (1만 이상 ~ 1억 미만)
fmtShares(500_000_000)  → "5억주"        (1억 이상)
fmtShares(1_234)        → "1,234주"      (1만 미만)
fmtShares(-150_000)     → "-15만주"      (음수는 부호 보존)
fmtShares(null)         → "-"
```

수량·주식수 컬럼에서 `fmt(n, 0)` 대신 사용한다.

### `cc(v)` — 부호별 색상 클래스

```
cc(양수)  → "text-red-600"   상승
cc(음수)  → "text-blue-600"  하락
cc(0/null) → "text-gray-400" 변동 없음
```

### 사용 패턴

```tsx
// 금액
<td>{fmt(price_krw, 0)}원</td>

// 부호 포함 변동
<td className={cc(diff)}>{diff > 0 ? "+" : ""}{fmt(diff, 0)}</td>

// 비중·변동(%)
<td className={cc(pct)}>{pct > 0 ? "+" : ""}{fmt(pct, 1)}%</td>

// 억원 단위
<td>{fmt(nation_value / 1e8, 0)}억원</td>
```

## 색상 코딩 (상승·하락)

| 상태 | 색상 | Tailwind 클래스 |
|------|------|----------------|
| 상승(+) | 빨강 | `text-red-600` |
| 하락(-) | 파랑 | `text-blue-600` |
| 변동 없음 / 해당 없음 | 회색 | `text-gray-400` |

## 차트 툴팁 (Recharts Tooltip)

모든 Tooltip에 아래 props를 일관되게 적용한다.

```tsx
contentStyle={{ fontSize: 12, padding: "5px 10px", border: "1px solid #e5e7eb", borderRadius: 6 }}
labelStyle={{ fontSize: 11, fontWeight: 600, color: "#374151", marginBottom: 2 }}
itemStyle={{ fontSize: 12, padding: "1px 0" }}
```

- 날짜 축을 가진 차트는 `labelFormatter={(l) => String(l)}` 으로 레이블 형식 명시
- 스파크라인(높이 50px 이하) 등 소형 차트는 툴팁 생략 또는 동일 스타일 적용

## 다크모드 미지원

이 앱은 다크모드를 지원하지 않는다.
`globals.css`에서 `prefers-color-scheme: dark` 블록을 제거해 OS 설정과 무관하게 항상 라이트모드 색상(`--foreground: #171717`, `--background: #ffffff`)을 유지한다.

다크모드 관련 CSS를 추가하지 않는다. 모든 화면·팝업의 텍스트 가시성은 이 전역 설정으로 보장한다.

## 레이아웃

- 모든 페이지 최대 너비: `max-w-7xl mx-auto` (AppLayout 내부 첫 번째 div)

- 테이블 헤더: `bg-gray-50 border-b border-gray-200`, 셀 `text-xs font-semibold text-gray-700`
- 테이블 행 hover: `hover:bg-gray-50`
